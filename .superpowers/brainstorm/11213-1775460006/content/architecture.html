System Architecture

DVC Fun Games — Standalone Next.js app on Vercel

Data Flow

Google Drive
Search "Week X 2026 - DVC Fun Games Registration"
→
Google Sheets
Game Schedule tab + registration tabs per game
→
API Route
/api/schedule?week=16
Server-side only
→
Dashboard UI
Client polls every 5 min, week nav via query param

Project Structure

dvc-schedule/
  src/app/
    page.tsx ← Dashboard (client component)
    layout.tsx ← Root layout, dark mode
    api/schedule/
      route.ts ← Fetches Sheets data, returns JSON
  src/lib/
    sheets.ts ← Google Sheets/Drive client + data parsing
    types.ts ← GameSchedule, Position, WeekInfo types
  src/components/
    game-card.tsx ← Single game card with position grid
    week-navigator.tsx ← Prev/next arrows, week label
    position-slot.tsx ← Individual position cell (green/orange/red)
    day-group.tsx ← Groups games under a date header

Key Logic

Position Needed Formula
Open Spiker = Teams × 2
Opposite = Teams × 1
Mid Blocker = Teams × 2
Setter = Teams × 1
Status Badge Logic
Cancelled → grey badge
All positions 0 → Full
Any position ≤2 → Almost full
Otherwise → Slots open
Filled Count
Count rows in registration tab where Position = "Open Spiker" etc.
Available = Needed − Filled
Cancellation
Admin adds "Status" column to Game Schedule sheet. Set value to "Cancelled". Dashboard reflects on next poll.
